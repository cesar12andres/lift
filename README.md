# Lift — workout tracker PWA

Local-first, tap-to-count workout tracker that lives inside this Obsidian vault. Hevy-compatible CSV format so exports slot straight into `raw-sources/workout_data.csv` and the wiki's existing `source-workout-data.md` / `fitness.md` pipeline.

## Features

- **Tap-to-count reps** — fullscreen overlay with a giant button; haptic on each tap; save as a set
- **Workout logging** — weight + reps per set, warmup / failure / drop set types, exercise notes
- **Rest timer** — floating countdown, +30 / skip, beep + vibration on end, configurable default
- **Templates** — save any workout as a template, start future sessions from it (3 seeded: Push / Pull / Legs)
- **Personal records** — auto-detected via est-1RM (Epley); PR badge + toast during logging
- **Volume tracking** — per workout, weekly totals, muscle-group frequency over 28 days
- **Exercise library** — ~80 common lifts seeded (matched to Hevy names); add custom; filter by muscle
- **Body weight** — quick log, chart over time, kg/lb toggle
- **Suggested next** — surfaces muscle groups not trained in the last 5 days
- **Exports** — Hevy-format CSV, JSON backup, Markdown summary (for wiki ingest)
- **Import** — Hevy CSV (seeds history + PRs instantly), JSON backup

All data lives in `localStorage`. Nothing leaves your device unless you export it.

## Running it

### Option 1 — On desktop (quick test)

Open `index.html` directly in Chrome or Safari. Most features work (`localStorage`, rest timer, imports, exports). The service worker only activates when served over `https://` or `file://` varies by browser — OK for a quick check either way.

### Option 2 — On iPhone (the actual use case)

You need the HTML served to Safari. Three paths, pick one:

**A. iCloud Drive / Obsidian Sync (easiest if you already sync the vault to iOS).**
1. Your vault is synced to iPhone (Obsidian Sync subscription or iCloud).
2. Open Files app → find `apps/workout/index.html` → tap to open in Safari.
3. Safari → Share → **Add to Home Screen**.
4. Tap the new icon — it launches fullscreen.

**B. GitHub Pages (most reliable).**
1. Push the vault (or just `apps/workout/`) to a GitHub repo.
2. Enable Pages on that repo.
3. Visit the URL on your iPhone → Share → **Add to Home Screen**.
4. Now the PWA works offline (service worker caches it after first visit) and is available even without the vault synced.

**C. Local Wi-Fi dev server (when testing changes).**
- Run any static server in this folder (e.g. `python -m http.server 8080`), then visit `http://<your-Mac-ip>:8080` from your iPhone on the same Wi-Fi.

### First-run: seed your existing Hevy data

Once loaded: **More → Import Hevy CSV** → pick `../../raw-sources/workout_data.csv`. All 16 sessions and ~428 rows will be imported with PRs / volume auto-computed.

## Vault sync workflow

The app is isolated from the filesystem (it's a browser-sandboxed PWA), so data flows out manually:

1. In the app: **More → Export CSV (Hevy format)** → saves `workout_data.csv`
2. On Mac/PC: move / AirDrop the file into `raw-sources/workout_data.csv` (overwrite)
3. Ask Claude in this vault: *"re-ingest workout_data.csv"* — the wiki's `source-workout-data.md` and `fitness.md` pages get updated with the new sessions, PRs, split analysis.

For lighter-weight vault updates, use **Export Markdown** — produces a ready-to-file MD summary you can paste into a wiki page.

For full backups: **Export JSON backup** (can be re-imported to restore all state).

## Tap-to-count mode

Tap the **🎯** button on any set → fullscreen counter → each tap on **TAP TO COUNT** adds a rep (with vibration) → **Save set** stores the count and marks the set done. Useful when:

- you're doing high reps and lose track
- you want to count without looking (put phone down, tap with knuckle through shirt, etc.)
- you're doing a drop set / AMRAP where the rep target isn't fixed

Manual entry still works — the tap counter is optional.

## Architecture notes

- Single HTML file with inline CSS + JS (no build step, no npm). Deliberately portable.
- State schema in `localStorage` under the key `lift.v1`.
- Exercise names match Hevy's naming (`Bench Press (Barbell)`, `Pendulum Squat (Machine)`, …) so import / export / PR comparisons line up.
- CSV columns exactly match the Hevy export header — the existing `wiki/source-workout-data.md` ingestion works unchanged.
- 1RM estimate via Epley: `weight × (1 + reps / 30)`. Used only to compare sets for PR detection; not displayed as a claim of literal 1RM.

## What's not included (deliberately)

- Cloud sync — would need a backend; data lives on device.
- Auto rep counting from motion sensors — unreliable for most lifts; not worth the false counts.
- Social / feed / sharing — personal tool.
- HealthKit integration — requires a native app.
