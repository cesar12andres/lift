# Lift — workout tracker PWA

Local-first, tap-to-count workout tracker that lives inside this Obsidian vault. Hevy-compatible CSV format so exports slot straight into `raw-sources/workout_data.csv` and the wiki's existing `source-workout-data.md` / `fitness.md` pipeline.

Designed around the coach loop: Claude writes next week's program to `programs/current.json`, the app fetches it on launch and renders today's planned session on the Home screen with a one-tap Start button.

## Features

- **Today's session, front and center** — the Home hero card shows the program-prescribed session for the current day. One tap pre-fills the active workout with the planned exercises, sets × reps, target loads, and RPE.
- **Program screen** — renders the full 7-day weekly plan published by the `/coach` skill: day type (strength / cardio / rest / conditioning), each exercise with sets × reps / target / RPE / rest / note, and a Start button per day.
- **Calendar heatmap** — GitHub-style grid on the Log screen, 16 weeks × 7 days. Violet = strength, amber = cardio, split = both. Tap any day → see the sessions.
- **Cardio / external session logging** — boxing, spinning, wrestling, running, walking, swimming, other. Fields: duration, kcal, avg HR, note, date. Counts toward streak + heatmap alongside strength.
- **Apple Health import** — import your Health app's `export.xml` (Settings → Health → profile → Export All Health Data → unzip → import `export.xml`). All `HKWorkout` entries are parsed into cardio sessions.
- **Tap-to-count reps** — fullscreen overlay with a giant button; haptic on each tap; save as a set.
- **Workout logging** — weight + reps per set, warmup / failure / drop set types, exercise notes.
- **Rest timer** — floating countdown, +30 / skip, beep + vibration on end, configurable default.
- **Templates** — save any workout as a template, start future sessions from it (3 seeded: Push / Pull / Legs).
- **Personal records** — auto-detected via est-1RM (Epley); PR badge + toast during logging.
- **Volume tracking** — per workout, weekly totals, muscle-group frequency over 28 days.
- **Exercise library** — ~80 common lifts seeded (matched to Hevy names); add custom; filter by muscle.
- **Body weight** — quick log, chart over time, kg/lb toggle.
- **Exports** — Hevy-format CSV, JSON backup, Markdown summary (for wiki ingest).
- **Import** — Hevy CSV (seeds history + PRs instantly), JSON backup, Apple Health XML.

All data lives in `localStorage` under `lift.v1`. The program JSON is fetched from `./programs/current.json` on every launch (network-first, with a cached fallback for offline). Nothing leaves your device except what you explicitly export.

## The coach loop

Weekly flow:

1. **Friday / Sunday** — run `/coach` in Claude inside this vault. Claude reads the last 7 days of training + nutrition + wiki context, diagnoses adherence + progression, and writes `apps/workout/programs/current.json` with next week's plan, plus a markdown brief in `wiki/coach-weekly-YYYY-MM-DD.md`.
2. `cd apps/workout && git add programs/ && git commit -m "coach: week of YYYY-MM-DD" && git push`
3. **Monday morning** — open the app. It fetches the fresh program in the background; within ~30s the Home hero card shows Monday's session from the new plan.

The program schema lives in `.claude/commands/coach.md` — it's what Claude writes to. The app reads it; both halves stay in sync.

### Overriding / forcing the coach

Pass arguments to `/coach` to steer:

- `/coach deload` — force a deload week
- `/coach push harder` — pull overload up (within safety margins)
- `/coach skip legs next week` — specific instruction, overridden only if it clashes hard with phase logic
- `/coach <free text>` — e.g. `/coach traveling Wednesday, need bodyweight-only alt`

## Running it

### Option 1 — GitHub Pages (the deployed path)

Already set up: `https://cesar12andres.github.io/lift/`. On iPhone: open in Safari → Share → **Add to Home Screen**. Works offline after first launch (service worker caches the shell). The program JSON fetches network-first so coach updates propagate within seconds.

### Option 2 — From the vault on iOS

1. The vault syncs to iPhone (Obsidian Sync or iCloud).
2. Files app → `apps/workout/index.html` → Open in Safari → Add to Home Screen.

Note: file:// loads can't fetch `./programs/current.json` via `fetch()` due to CORS — use Option 1 if you need live program updates.

### Option 3 — Local Wi-Fi dev server (for development)

```sh
cd apps/workout
python -m http.server 8080
```

Visit `http://<your-ip>:8080` from iPhone on same Wi-Fi.

### First-run: seed existing history

**More → Import Hevy CSV** → pick `../../raw-sources/workout_data.csv`. All sessions import with PRs / volume auto-computed.

## Apple Health integration

PWAs cannot access HealthKit directly — that requires a native iOS app. Workaround: import the Health app's full export.

**One-time:**

1. On iPhone, open **Health** → tap your profile picture (top right) → **Export All Health Data**.
2. Wait for the export (~1-5 min for multi-year data). Share sheet opens.
3. Save `export.zip` somewhere you can unzip (Files app can unzip).
4. In Lift: **More → Import Apple Health XML** → pick the unzipped `export.xml`.

All `<Workout>` entries (boxing, spinning, running, walking, etc.) are parsed into the cardio log with duration + kcal. Heart rate requires a separate shortcut — not yet wired.

**Automating it (optional):**

Apple **Shortcuts** can generate a simplified log automatically. Create a Shortcut that finds recent workouts → formats as CSV → saves to Files / iCloud Drive. You can then periodically import that file. A ready-made recipe will ship here later.

## Vault sync workflow

1. **More → Export CSV (Hevy format)** → saves `workout_data.csv`.
2. Move / AirDrop the file into `raw-sources/workout_data.csv` (overwrite).
3. Ask Claude: *"re-ingest workout_data.csv"* — `source-workout-data.md` and `fitness.md` get updated.

For vault-ready markdown: **More → Export Markdown**.
For full backups: **More → Export JSON backup** (can be restored via Import JSON).

## Tap-to-count mode

Tap the **🎯** button on any set → fullscreen counter → each tap on **TAP TO COUNT** adds a rep (with vibration) → **Save set** stores the count and marks the set done.

## Architecture notes

- Single HTML file with inline CSS + JS (no build step, no npm). Deliberately portable.
- State schema in `localStorage` under the key `lift.v1`. Program JSON cached separately under `lift.program`.
- Exercise names match Hevy's naming so import / export / PR comparisons line up.
- CSV columns exactly match the Hevy export header — the existing wiki ingestion works unchanged.
- 1RM estimate via Epley: `weight × (1 + reps / 30)`. Used only for PR detection.
- Service worker: cache-first for app shell; network-first for `programs/*` so coach updates propagate fast.
- Design: violet/indigo gradient (`#8b5cf6 → #6366f1`) for strength, amber/red gradient for cardio.

## What's not included (deliberately)

- Cloud sync for workouts — would need a backend; data lives on device.
- Auto rep counting from motion sensors — unreliable for most lifts.
- Social / feed / sharing — personal tool.
- Direct HealthKit integration — requires a native app. XML import is the supported path.

If any of these matter, open a chat with Claude and we extend.
